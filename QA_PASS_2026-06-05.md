## 🔴 CRITICAL — cross-cutting bug found during QA: `apiRequest` argument order

**What:** Commit `59afb04` ("Improve email setup process…") flipped the `apiRequest` signature from
`apiRequest(method, url, data)` → `apiRequest(url, method, data)` (client/src/lib/queryClient.ts:38)
but did **not** update the existing call sites. Every call written *before* that commit still passes
method-first, e.g. `apiRequest('POST', '/api/...', data)`. With the new signature that becomes
`fetch('POST', { method: '/api/...' })` → an **invalid HTTP method**, so the request rejects and the
button silently does nothing (or shows a generic error toast).

**Why it's hidden:** core auth bypasses `apiRequest` — real login/register/logout use raw `fetch` in
`hooks/use-auth.tsx`. And `Login.tsx` (which has the broken call) is orphaned — not in the router
(`/auth` → `auth-page.tsx`). Newer voice-era code (Contacts, forms) uses the correct `(url, method)` order.
So the app "works" on the paths you use most, masking a whole class of dead buttons elsewhere.

**Scope — broken mutation call sites on still-routed surfaces:**
- `pages/Dashboard.tsx` — toggle task (:228), create campaign (:242)
- `pages/Settings.tsx` — generate tracking code (:734)
- `pages/SystemNotifications.tsx` — mark read / delete / mark-all-read (:95,:119,:143)
- `pages/LeadManagement.tsx` — add note (:204), assign owner (:229)
- `pages/FeedbackList.tsx` — load notifications (:41)
- `components/leads/WorkflowAutomation.tsx` — create / toggle / test workflow (3)
- `components/leads/LeadScoringAlgorithm.tsx` — save config / recalculate (2)
- `components/leads/CustomerSegmentation.tsx` — create segment / export (2)
- `components/campaigns/MessageVariants.tsx` — create A/B variant (1)
- `components/ai/ChatAssistant.tsx` — send chat message (1)
- `components/feedback/FeedbackButton.tsx` — submit feedback (1)
- `pages/Customers.tsx` — create customer (1) *(legacy page, may be unrouted — confirm)*

Dead/ignore: `Login.tsx` (orphaned), `Dashboard.tsx.new` (cruft).

**Recommended fix (one file, low-risk):** make `apiRequest` tolerate both orders — if the first arg is a
bare HTTP verb (`GET|POST|PUT|PATCH|DELETE`) and the second looks like a path, swap them. This un-breaks
every old call site at once without editing ~12 files, and new `(url, method)` calls keep working (a real
URL never equals a bare verb). Safer than hand-editing each site. Then optionally migrate call sites later.

**STATUS: ✅ FIX APPLIED 2026-06-05** to `client/src/lib/queryClient.ts` (backward-compat arg swap). Awaiting
diff review + Replit publish to verify live. Local `tsc` not runnable in this env (typescript not in node_modules);
will be checked by Replit's `npm run check`/build.
